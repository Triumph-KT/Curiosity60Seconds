# MVP Requirements Specification
## Knowledge Externalization Platform

**Version:** 1.0
**Author:** Triumph Kia Teh
**Date:** April 2026

---

## 1. Product Overview

A multi-user knowledge publishing platform where users paste raw research (links, text, images, AI-generated answers), and an AI model synthesizes it into a cited, structured blog post written in the user's personal voice. The post publishes instantly to a public-facing profile page indexed by Google. The platform is built for multiple users from day one, with each user owning a private workspace and a public-facing profile.

---

## 2. Core User Workflow

1. User pastes a question and raw sources (text, links, images with captions) into the input page
2. AI synthesizes inputs into a structured blog post with images woven inline and citations at the bottom, written in the user's voice
3. Post saves to the database and publishes instantly to a public URL
4. Anyone with the URL can read the post. Google can index it.
5. User can edit and republish any post at any time

## 4. Database Schema

The platform requires a minimum of four database tables. All tables must support full CRUD operations where applicable. An audit log must record all create, update, and delete events automatically via database triggers.

### 4.1 users
| Column | Type | Notes |
|---|---|---|
| id | uuid | Primary key |
| email | text | Unique, required |
| name | text | Display name |
| username | text | Unique, used in public URLs |
| photo_url | text | Profile photo, stored in Supabase Storage |
| bio | text | Short user bio |
| writing_samples | text[] | Three past writing samples collected at onboarding |
| voice_prompt | text | AI-generated system prompt derived from writing samples and preferences |
| role | enum | 'user' or 'admin' |
| daily_generation_count | integer | Number of AI generations performed today, resets at midnight UTC |
| daily_generation_limit | integer | Max Claude API generations per day, default 10 |
| created_at | timestamp | Auto-set |

### 4.2 posts
| Column | Type | Notes |
|---|---|---|
| id | uuid | Primary key |
| user_id | uuid | Foreign key to users |
| title | text | Generated by AI, editable |
| slug | text | SEO-friendly URL segment, unique per user |
| body_md | text | Full generated Markdown of the post, rendered to HTML at display time |
| raw_input | jsonb | Original question, sources, and image references submitted by user |
| status | enum | 'draft' or 'published' |
| published_at | timestamp | Set on first publish |
| created_at | timestamp | Auto-set |
| updated_at | timestamp | Auto-updated |

### 4.3 sources
| Column | Type | Notes |
|---|---|---|
| id | uuid | Primary key |
| post_id | uuid | Foreign key to posts |
| url | text | Source URL if applicable |
| quote | text | Quoted text from the source |
| image_url | text | Stored image URL if source is an image |
| image_caption | text | Caption provided by user |
| source_label | text | Display label for citation |
| created_at | timestamp | Auto-set |

### 4.4 audit_log
| Column | Type | Notes |
|---|---|---|
| id | uuid | Primary key |
| user_id | uuid | Who performed the action |
| action | text | 'created', 'updated', 'deleted', 'viewed' |
| table_name | text | Which table was affected |
| record_id | uuid | Which record was affected |
| metadata | jsonb | Additional context |
| timestamp | timestamp | Auto-set |

The audit_log must be populated automatically via Postgres triggers on insert, update, and delete events across the posts and users tables. Manual writes to audit_log are not required.

## 5. Pages and Routes

### 5.1 Public Pages (no authentication required, Google-indexable)

**`/`** — Landing page. Explains the platform. Includes sign up and login call to action.

**`/people`** — Shared user directory. Displays all registered users with profile photo and display name. Each user card links to their public profile. This is the only page where users can see each other.

**`/u/[username]`** — Public profile page. Displays the user's photo, name, bio, and a list of all their published posts sorted by date. Readable by anyone.

**`/u/[username]/[slug]`** — Full post page. Displays the complete published post with title, author name and photo, publish date, body with images woven inline, and a citations section at the bottom. This is the page Google indexes.

### 5.2 Authentication Pages

**`/signup`** — Account creation. Collects email and password. On success, redirects to `/onboarding`.

**`/login`** — Sign in. On success, redirects to `/dashboard`. If user has not completed onboarding, redirects to `/onboarding`.

### 5.3 Onboarding Page

**`/onboarding`** — Runs exactly once per user after first signup. Collects:
- Display name and username
- Profile photo upload
- Short bio
- Three past writing samples (from any platform: social media, essay, journal entry)
- Preference questions (tone, topics of interest, writing style)

On submission, the platform sends the writing samples and preferences to the Claude API with a system prompt instructing it to produce a concise voice prompt for this user. The output is saved to `users.voice_prompt`. The user's `onboarded` flag is set to true. User is redirected to `/dashboard`.

### 5.4 Private Pages (authentication required)

**`/dashboard`** — User's private home. Lists all their posts (drafts and published) with title, status, date, and links to view or edit. Includes a button to create a new post.

**`/new`** — Core input page. Contains:
- A text field for the user's question
- A large text area to paste raw sources (text from Perplexity, Google, AI tools, copied articles)
- An image uploader that accepts multiple images, each with a source label and caption field
- A generate button

On submit, the system executes the generation and publishing pipeline (see Section 6).

**`/post/[id]/edit`** — Edit page for any post. Pre-fills with existing content. User can modify the raw input or the generated body directly. On save, the post is republished with an updated timestamp.

**`/settings`** — Profile settings. User can update name, bio, photo, and writing samples. Updating writing samples triggers a regeneration of the voice prompt.

### 5.5 Admin Page (admin role only)

**`/admin`** — Full platform view. Displays:
- List of all registered users with account details
- Ability to view any user's posts
- Ability to deactivate or delete user accounts
- Audit log viewer showing all recorded actions across the platform

## 6. AI Generation and Publishing Pipeline

This pipeline executes when a user submits the `/new` form.

**Step 1:** Save raw input to the posts table with status 'draft'.

**Step 2:** Upload all submitted images to Supabase Storage. Retrieve their public URLs.

**Step 3:** Save each source (URLs, quotes, images) to the sources table linked to the post.

**Step 4:** Construct the AI prompt:
- Prepend the user's `voice_prompt` as the system prompt
- Include the user's question
- Include all pasted text sources
- Include all image public URLs with their captions and source labels
- Instruct the model to produce a complete Markdown document with: a title, structured sections, images placed inline at relevant points using standard Markdown image syntax `![caption](url)` with source label included in the caption, and a citations section at the bottom linking back to all original sources

**Step 5:** Check the user's `daily_generation_count` against `daily_generation_limit`. If the user is under their daily limit, send to Claude API and increment `daily_generation_count`. If the daily limit is reached, route automatically to Ollama local model with the same prompt. No error is shown to the user. If the Claude API call fails for any reason, also fall back to Ollama silently.

**Step 6:** Save the returned Markdown to `posts.body_md`. Set `posts.status` to 'published'. Set `posts.published_at` to current timestamp. Generate the SEO slug from the post title.

**Step 7:** Redirect the user to the live public post URL: `/u/[username]/[slug]`.

## 7. SEO Requirements

Every public post page must export the following metadata dynamically via Next.js App Router metadata API:
- `title` — post title
- `description` — first 160 characters of post body stripped of HTML
- `canonical` — full public URL of the post
- `og:title`, `og:description`, `og:image` — Open Graph tags for social sharing
- `author` — user's display name

Post slugs must be human-readable and generated from the post title. Example: `/u/triumph/why-trump-posted-himself-as-jesus-on-twitter`

## 8. Access Control Rules

| Page | Public | Authenticated User | Admin |
|---|---|---|---|
| Landing `/` | Yes | Yes | Yes |
| People directory `/people` | Yes | Yes | Yes |
| Public profile `/u/[username]` | Yes | Yes | Yes |
| Public post `/u/[username]/[slug]` | Yes | Yes | Yes |
| Dashboard `/dashboard` | No | Own only | Yes |
| New post `/new` | No | Yes | Yes |
| Edit post `/post/[id]/edit` | No | Own only | Yes |
| Settings `/settings` | No | Own only | Yes |
| Admin `/admin` | No | No | Yes |

Users cannot view, access, or modify any other user's private data, drafts, or account settings. The only shared visibility between users is the people directory and public post pages.

---

## 9. Image Handling Requirements

- Users can upload multiple images per post
- Each image must have a caption and a source label provided by the user
- Images are stored in Supabase Storage and served via public URL
- Images must appear inline in the published post body at contextually relevant positions as determined by the AI
- Each inline image must display its caption and source label beneath it
- All images must also appear in the citations section with their source label and original URL if provided

## 10. Build Order

Build in this exact sequence. Do not proceed to the next phase until the current phase is tested and working.

1. Scaffold Next.js app with Tailwind CSS and Supabase client configured
2. Create all four database tables in Supabase with correct schema and relationships
3. Implement Postgres triggers for audit_log on posts and users tables
4. Build authentication: signup, login, session handling, route protection
5. Build onboarding page and voice prompt generation via Claude API
6. Build the `/new` page: form, image upload to Supabase Storage, full AI generation pipeline including Claude API integration and Ollama fallback, daily generation limit check, post save and publish in Markdown
7. Build the public post page `/u/[username]/[slug]` with SEO metadata
8. Build the public profile page `/u/[username]`
9. Build the people directory `/people`
10. Build the dashboard `/dashboard`
11. Build the edit page `/post/[id]/edit`
12. Build the settings page `/settings`
13. Build the admin panel `/admin`
14. Configure Vercel deployment connected to GitHub repo
15. End-to-end test: create account, onboard, publish a post, verify it is publicly accessible and SEO metadata is correct

---

## 11. Out of Scope for MVP

The following features are explicitly excluded from this version and reserved for future builds:

- Browser extension for capture
- One-click cross-platform publishing (LinkedIn, X, newsletter)
- Topic hub pages clustering posts by question
- Community comments and discussions
- Follow system between users
- Monetization layer
- Mobile app
- Voice memo input
- Living document update diffs
- Multi-author collaborative posts
